Задание на практику для реализации криптоалгоритма SOSEMANUK.
Инициализация происходит в два этапа:
1) Задается расписание ключей (key schedule). Секретный ключ не зависит от начального значения.
Для этого используется примитв Serpent24, который предоставляет 25 128-битных раундовых ключей.
2) Вводится начальное значение (IV injection). Для этого используется заданное расписание ключей и IV.
В качестве IV выступают выходные значения двенадцатого, восемнадцатого и двадцать четвертого раудна Serpent24.

Работа конечного автомата. SOSEMANUK использует конечный автомат  с 64-битами памяти, что соответсвует
двум 32-битным регистрам R1 и R2. Автомат принимает на вход несколько значений, полученных с помощью LFSR,
обновляет регистры, после чего выдает 32-битное значение.

Преобразование данных. Данные, полученные с помощью LFSR, поступают на вход FSM, где
происходит их нелинейное преобразование. Затем к четырем выходным значениям конечного автомата применятеся
табличная замена (S-box) и XOR с соответствующими значениями регистра сдвига.
